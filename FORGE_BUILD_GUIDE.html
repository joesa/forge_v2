Step 2 — Create your Inngest account

Inngest replaces Trigger.dev entirely. It is event-driven: your app sends events, Inngest executes the corresponding functions durably with automatic retries. A local dev server lets you test everything before deploying.
Inngest setup checklist
Create account at inngest.com
Create an app named "forge"
Settings → Event keys → copy Event Key → INNGEST_EVENT_KEY
Settings → Signing keys → copy Signing Key → INNGEST_SIGNING_KEY
For local dev: use INNGEST_EVENT_KEY=local-dev and INNGEST_SIGNING_KEY=signkey-dev-000000000
Install Inngest CLI: npm install -g inngest-cli (used for local dev server)
Local dev workflow: When running locally, start a third terminal with npx inngest-cli@latest dev -u http://localhost:8000/api/inngest. This starts a dev UI at localhost:8288 where you can trigger events manually and watch functions execute in real time.

Supabase Auth REST · register MUST write User row to Supabase PostgreSQL · login safety net
Two-step pattern on every register AND login. Supabase Auth stores the identity. Your public.users table stores application data. The Supabase user UUID is used as our User.id. If register_user() skips the get_write_session() call, get_current_user() always returns None.

Build the auth service and API using Supabase Auth.

backend/app/services/auth_service.py:

  Auth REST base: {SUPABASE_URL}/auth/v1

  Admin headers: {"apikey": SERVICE_ROLE_KEY, "Authorization": "Bearer {SERVICE_ROLE_KEY}"}
  Anon headers: {"apikey": ANON_KEY, "Content-Type": "application/json"}

  class SupabaseAuthError(Exception):
    def __init__(self, status_code: int, detail: str)

  async def register_user(email, password, display_name) -> dict:
    Step 1: POST /auth/v1/signup with admin headers
      body: {"email", "password", "data": {"display_name": display_name}}
    Raise SupabaseAuthError if status >= 400
    Step 2: Write User row via get_write_session():
      User(id=UUID(response["user"]["id"]), email=email,
           display_name=display_name, onboarded=False, plan="free")
      Duplicate guard: check existing before insert.
    Return response dict.

  async def get_or_create_user_on_login(supabase_uid, email, display_name) -> User:
    Idempotent: check if User row exists, create if not. Uses get_write_session().

  async def login_user(email, password) -> dict:
    POST /auth/v1/token?grant_type=password with anon headers
    Raise SupabaseAuthError if status >= 400
    Call get_or_create_user_on_login() with returned user data.
    Return response dict (contains access_token, refresh_token, user).

  async def refresh_tokens(refresh_token) -> dict:
    POST /auth/v1/token?grant_type=refresh_token

  async def get_current_user(user_id: UUID, session: AsyncSession) -> User | None:
    SELECT from public.users WHERE id=user_id via caller-supplied read session.

  async def logout_user(access_token) -> None:
    POST /auth/v1/logout, ignore 401 (already logged out).

  async def forgot_password(email) -> None:
    POST /auth/v1/recover. Swallow ALL 4xx (prevent email enumeration). Raise on 5xx only.

  async def reset_password(token, new_password) -> None:
    PUT /auth/v1/user with Bearer token.

    trigger=inngest.TriggerEvent(event="inngest/function.failed"),
)
async def pipeline_failure_handler_fn(ctx: inngest.Context, step: inngest.StepAsync) -> None:
    """
